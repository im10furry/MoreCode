# MoreCode 自我迭代系统设计文档

## 概述

本系统为 MoreCode 项目设计了一套完整的 7x24 小时自我迭代机制，允许 AI 助手自主发现问题、修复错误和持续改进。

## 工作流程

### 1. 健康监控循环
- 定期运行代码检查 (`cargo clippy`, `cargo fmt`)
- 执行单元测试和集成测试
- 监控构建状态
- 收集性能指标

### 2. 问题发现机制
- **代码质量问题**: 通过 Clippy 和自定义规则发现
- **测试失败**: 自动分析失败的测试用例
- **性能问题**: 监控执行时间和资源使用
- **文档过期**: 检测代码与文档的不一致

### 3. 自我修复流程
1. 发现问题 → 2. 创建修复任务 → 3. Agent 协作修复 → 4. 测试验证 → 5. 自动提交

### 4. 持续改进流程
- 定期分析代码库，寻找优化机会
- 基于使用模式自动改进 Agent 提示词
- 更新文档和示例

## Token 预算管理

### 预算分配策略
- **健康监控**: 每日 10% 预算
- **问题修复**: 每日 50% 预算
- **持续改进**: 每日 30% 预算
- **预留应急**: 每日 10% 预算

### 时间分配
- 低峰时段 (深夜): 执行深度代码分析和大规模重构
- 正常时段: 执行常规检查和小修复
- 根据 `quiet_hours` 配置自动调整

## 配置示例

```toml
# ~/.morecode/daemon.toml
[daemon]
enabled = true
profile = "full-extensible"
daily_budget_usd = 20.0

[self_iteration]
enabled = true
health_check_interval_mins = 30
auto_fix_enabled = true
auto_commit_enabled = true
max_daily_tasks = 20

[self_iteration.budget_allocation]
health_monitoring = 0.10
bug_fixes = 0.50
improvements = 0.30
contingency = 0.10

[self_iteration.rules]
check_clippy = true
check_tests = true
check_docs = true
suggest_refactoring = true
```

## GitHub Actions 集成

### 自动触发工作流
- 定时运行: 每 4 小时检查一次
- PR 自动审查: 对所有 PR 进行 AI 审查
- Issue 自动分析: 分析新 Issue 并生成修复建议

### 安全措施
- 所有自动提交需要通过 CI 检查
- 关键代码修改需要人工审核
- Token 使用实时监控和告警

## 自我迭代任务类型

### 1. 健康检查任务
```
instruction = "运行 cargo clippy --workspace -- -D warnings，报告所有警告"
priority = "high"
schedule = "every_30_minutes"
```

### 2. 测试修复任务
```
instruction = "分析并修复失败的测试用例：{test_names}"
priority = "critical"
schedule = "on_demand"
```

### 3. 代码优化任务
```
instruction = "审查并优化 {file_path} 的性能和可读性"
priority = "medium"
schedule = "during_quiet_hours"
```

### 4. 文档更新任务
```
instruction = "检查并更新与 {module} 相关的文档"
priority = "low"
schedule = "weekly"
```
